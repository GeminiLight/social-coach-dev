# Case Study Text

两个场景，展示 Agentic Consensus 如何处理真实任务。
每个场景结构相同：背景设定、vibe coding 的盲区、C 层如何处理。

---

## Case B — ML 训练流水线：静默性能退化

**背景。**
推荐模型的周度评估 AUC 下降了 2 个点。
`click_rate_7d` 是该模型按特征重要性排名第一的特征。
工程师通过 chat 驱动的调试连续尝试了三天：调学习率、回滚特征、加 null check 补丁，均无效。
每次尝试都在不知道前几次已经排除了什么的情况下重新开始。

**Vibe coding 的盲区。**
所有假设只存在于对话历史中。
每次新会话，agent 没有任何记录：哪些解释已经被排除、上周用的是哪个数据快照、特征分布是否在两周之间发生了变化。
根因是：特征流水线中 `click_rate_7d` 的计算窗口被悄悄从 7 天改成了 3 天——这是一个与该特征无关的 PR 的副作用。
这个变化不可见，因为没有任何结构将特征定义与使用该特征的模型关联起来。

**C 层如何处理。**
$C$ 维护一张流水线溯源图：`data_snapshot → feature_pipeline → training_config → model → eval_metrics`。
AUC 下降时，系统在 $C$ 中构造两个竞争假设，作为一等公民节点，每个节点明确列出所需的判别证据：

- 假设 A（数据漂移）：上游超时导致 wk12 的 `user_activity` 行数少于 wk11。所需证据：两个快照的行数对比。
- 假设 B（特征漂移）：`click_rate_7d` 的分布因流水线变更而发生偏移。所需证据：wk11 vs wk12 的分布 drift test。

由于 $\mathcal{H}(C \mid I)$ 很高，系统不猜测，而是请求这两个定向检查。
两个检查自动执行。数小时内得到结果：假设 A 被排除（快照行数完整）；假设 B 被确认（drift test $p < 0.001$，$\Psi$ 追溯到特征流水线的一次提交，该提交将窗口参数作为副作用修改，关联到 PR #847，而该 PR 的原始目的与此无关）。

修复最小且有证据支撑：将窗口回滚为 7 天，重新训练。AUC 当天恢复。

**收益。**
Vibe coding 用了 4 天、浪费了 3 个训练周期，根因仍然未知。
Agentic Consensus 在数小时内定位根因并完成修复，当天恢复 AUC。
根因和修复路径被写回 $C$：下次任何人提议修改特征窗口参数时，系统会自动将这次事故作为历史证据呈现，干预成本随时间递减。

## Case C — A/B 实验：特征泄漏

**背景。**
一个新推荐策略通过 A/B 测试评估。
分析结果：CTR lift +3.1%，p = 0.003，统计显著。
Agent 建议全量上线。

**Vibe coding 的盲区。**
实验组和对照组的特征流水线定义在另一个服务仓库中，不在 agent 的上下文范围内。
实验组有 13 个特征，对照组有 12 个。
多出来的特征 `user_realtime_signal` 只在实验组中存在——这意味着 lift 衡量的是特征优势，而不是策略优势。
统计检验在错误的输入上被正确地执行了；agent 没有任何办法发现这一点。

**C 层如何处理。**
$C$ 维护一张实验因果 DAG：`experiment_config → {control_group, treatment_group} → {feature_pipeline_ctrl, feature_pipeline_treat} → metric_definition`。
分析运行之前，auditor 检查两条流水线之间的特征一致性合约。
检查失败：实验组有 `user_realtime_signal`，对照组没有。
分析在产出任何数字之前被阻断。
    
工程师将 `user_realtime_signal` 从实验组移除，重新跑实验。
7 天后：CTR lift +0.4%，p = 0.31，不显著。该策略无效。

**收益。**
没有 $C$ 的情况下，这个实验会以"显著有效"的结论写入实验记录，触发全量上线，并在生产环境中消失 2--3 周后才暴露问题。
有了 $C$，错误在产出任何数字之前就被拦截：节省了 3 周的无效上线周期，避免了一条错误的实验结论污染历史记录。
原始结果（+3.1%，p = 0.003）在统计上是正确的，但在因果上是无效的；换一个更强的模型会产生完全相同的错误数字。
失败不在于生成质量，而在于结构，而结构只有 $C$ 能编码。

## 两个 case 共同说明的事情

Case B 展示 $C$ 作为**诊断界面**：竞争假设是显式的，不确定性可测量，证据收缩不确定性——而不是对话轮次收缩不确定性。

Case C 展示 $C$ 作为**有效性关卡**：结构合约在结果产出之前被检查，捕捉到任何统计复杂度都无法发现的错误。

两个 case 共同覆盖了 vibe coding 的两种主要失败模式：来自不可见上游变更的静默退化，以及来自结构无效实验的虚假置信。
两种情况下，失败都不是因为模型生成了错误的代码——而是因为主要 artifact（代码加对话）没有编码任何结构，而这些结构本可以捕捉到这个错误。

更广泛地看，这两个 case 揭示的不是两个孤立的工程失误，而是同一个系统性风险：随着 AI 生成能力不断提升，人类对系统的实际理解和控制能力正在同步下降。
模型越强，生成速度越快，opacity 积累得越快，单次失误的潜在影响也越大。
Agentic Consensus 的意义不只是"帮你更快找到 bug"——而是在 AI 吞吐量持续扩大的趋势下，为人类保留一个可审计、可干预、可追溯的控制界面。
没有这个界面，规模化的 AI 辅助工程最终会退化为规模化的不透明。
